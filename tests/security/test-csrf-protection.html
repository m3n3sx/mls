MASE CSRF Protection Test Suite

Purpose: Test for Cross-Site Request Forgery (CSRF) vulnerabilities

Requirement: 14.5 - Test for CSRF vulnerabilities

⚠️ Warning: These tests should only be run in a development environment. Do not run CSRF tests on production sites.

Test 1: AJAX Request Without Nonce

Endpoint: wp-admin/admin-ajax.php?action=mase_save_settings

Method: POST

Expected Result: Request should be rejected with 403 error

Test 2: AJAX Request With Invalid Nonce

Endpoint: wp-admin/admin-ajax.php?action=mase_save_settings

Method: POST with invalid nonce

Expected Result: Request should be rejected with 403 error

Test 3: AJAX Request With Expired Nonce

Endpoint: wp-admin/admin-ajax.php?action=mase_save_settings

Method: POST with expired nonce

Expected Result: Request should be rejected

Note: Nonces expire after 24 hours by default

Test 4: Cross-Origin Request

Scenario: Attacker site tries to make request to your WordPress site

Expected Result: Request should be blocked by CORS policy or nonce verification

Note: This requires setting up a separate test domain

Test 5: State-Changing GET Request

Scenario: Try to change settings via GET request

Expected Result: All state-changing operations should require POST

Manual Testing Instructions

  1. Open browser developer tools (F12)
  2. Go to Network tab
  3. Click each "Run Test" button above
  4. Observe the response status codes
  5. Verify that requests without valid nonces are rejected
  6. Check that error messages don't leak sensitive information

CSRF Protection Checklist

  • ✓ All AJAX endpoints verify nonces
  • ✓ All AJAX endpoints check user capabilities
  • ✓ Nonces are generated with wp_create_nonce()
  • ✓ Nonces are verified with check_ajax_referer()
  • ✓ State-changing operations require POST method
  • ✓ No sensitive operations via GET requests
  • ✓ Proper error handling without information leakage

Expected Nonce Verification Flow

1. Client requests page
2. Server generates nonce: wp_create_nonce('mase_save_settings')
3. Nonce included in JavaScript: maseAdmin.nonce
4. Client makes AJAX request with nonce
5. Server verifies nonce: check_ajax_referer('mase_save_settings', 'nonce')
6. If valid: Process request
7. If invalid: Return 403 error
